### Instrucciones generales
Cada estudiante deberá crear su carpeta donde alojará sus evidencias. 
La carpeta será nombrada con 1 nombre y 1 apellido, ejemplo: fulanito-perez. 
Dentro de la carpeta se creará una carpeta nueva con el nombre de "parcial2"

### 🚀 El Flujo de Trabajo (Workflow)
Antes de dar por terminada una tarea, asegúrate de haber cumplido este ciclo en tu terminal:
1.  `git pull origin main` (Para tener los últimos cambios del profesor).
2.  `git add .` (Para preparar tus archivos).
3.  `git commit -m "Entrega: Práctica XX - [Tu nombre y apellido]"` (Mensaje claro).
4.  `git push origin main` (Para subirlo a tu nube).
5. **Abrir Pull Request:** Solicita la revisión en la plataforma de GitHub.

### ⚠️ Reglas Anti-Errores
* **Nunca** modifiques archivos fuera de tu carpeta personal.
* **Nunca** subas archivos ejecutables pesados o carpetas de configuración (como `.vscode` o `__pycache__`).
